Foundation · อ่านก่อน Day 1

TouchDesigner คืออะไร

พื้นฐานที่ต้องเข้าใจก่อนลงมือ + ตารางเทียบกับเครื่องมือที่คุ้นเคย

TD คือ real-time node-based dataflow engine — วาง graph ของการแปลงข้อมูล แล้ว engine รันให้ทุกเฟรม เฉพาะจุดที่จำเป็น (ไม่ต้องเขียน render loop เอง). การทำงานคล้าย spreadsheet / React render tree / RxJS

TD ใช้ภาษา / เทคโนโลยีอะไร

engine + operator ทุกตัวเขียนด้วย C++ รันบน GPU — งานหนัก (ภาพ, signal, 3D, render) ทำในชั้นนี้. ภาษาที่ใช้ร่วม (scripting) คือ Python ที่ฝังมาในตัว (embedded) — ไว้ทำ logic, control, glue. ทำงานส่วนใหญ่ได้โดยไม่ต้องเขียน Python แค่ต่อ node + กรอกค่า; งานจริงจังค่อยใช้ Python ช่วย. นอกจากนี้เขียน GLSL (shader บน GPU) ได้ และต่อ custom operator ด้วย C++ ได้

rule of thumb: node สำหรับ dataflow/signal · Python สำหรับ logic, control flow, event, state, orchestration

หลักที่ dev ต้อง internalize

  • Cook = pull-based / lazy. node จะ cook เมื่อมี 2 อย่าง: (1) มีคนปลายทางขอ data (viewer เปิด / output node / export) และ (2) มีเหตุให้ cook (input/param เปลี่ยน หรือเป็น time-dependent เช่น noise/วิดีโอ/เสียง). → กิ่งที่ไม่มีใครใช้ = ฟรี. (เหมือน spreadsheet คำนวณเฉพาะ cell ที่กระทบ / React reconcile เฉพาะที่เปลี่ยน)
  • Operator = typed data families: TOP=ภาพ/GPU · CHOP=สัญญาณ/ตัวเลข stream · SOP=3D geo · DAT=text/table/script · COMP=container/module. ต่อภายใน family เดียวกัน ข้าม family ต้องมี operator แปลง
  • Reactive ไม่ใช่ imperative — ไม่สั่ง "ทำ 1 แล้ว 2" แต่ ประกาศความสัมพันธ์ ("รัศมีวงกลม = ระดับ bass") แล้ว engine รักษาให้จริงตลอด = data binding

เทียบ p5.js — draw loop

p5 คุณ เขียน loop เอง: function draw(){ r=bass*200; ellipse(x,y,r,r) } รันทุกเฟรม. TD ไม่มี draw() ที่คุณเขียน — cook cycle คือ loop เอง. คุณวาง graph: Audio In → Analyze(bass) → Math(*200) → (bind) Circle → Render. p5 รันทุกบรรทัดทุกเฟรมไม่ว่าจะเปลี่ยนไหม; TD cook เฉพาะ node ที่ "มีคนขอ + มีเหตุ" = ขี้เกียจกว่าโดย default

mindset ที่ต้องพลิก + dev gotcha

  • ไม่มี main() / entry point — graph + cook loop คือโปรแกรม · อยากให้อัปเดตทุกเฟรม = ทำ node ให้ time-dependent (ไม่เขียน loop)
  • ไม่มี if/else node — branch คือ เลือก data ที่ไหล (Switch) ไม่ใช่กระโดดคำสั่ง
  • state อยู่ใน operator/Storage ไม่ใช่ตัวแปร Python module-level (อันนั้น reset ได้ ไม่เข้ากับ dataflow)
  • "ทุก node รันทุกเฟรม" = ผิด (pull-based) · "เปิด viewer = ฟรี" = ผิด (สร้าง cook request — ปิด viewer ที่ไม่ใช้ช่วย FPS)
  • งานภาพ/signal อย่าดึงมาวนใน Python (ช้า) เก็บบน node (GPU) · ทำ .exe ไม่ได้ — ship ไฟล์ .toe

References / Exports — ต่อค่าข้าม operator (gap · สอนก่อนเพื่อน)

2 วิธีต่อค่าจาก operator นึงไปคุม parameter ของอีกตัว โดยไม่ลากเส้น. ข้อต่างที่ต้องจำ: Reference ใส่สูตรได้ · Export ใส่สูตรไม่ได้ (ส่งค่าดิบล้วน)

Reference (Expression)Export
ทำยังไงพิมพ์ Python expression ลงใน parameterลาก CHOP channel มาวางทับ parameter

เทียบ dev: Reference = computed getter get radius(){ return audio.low*2 } (ใส่สูตรได้) · Export = ต่อ stream เข้า prop ตรงๆ audio.low$.subscribe(v=>circle.radius=v) (ค่าดิบ)

parameter มี 4 โหมด ดูจากสีช่อง: Constant=เทา · Expression(ref)=ฟ้า · Export=เขียว · Bind=ม่วง

Logic / conditionals — ทำไมหา if/else ไม่เจอ (gap · dev ถามแน่)

Paradigm: ภาษาทั่วไป if/else = คุมว่าโค้ดบรรทัดไหนรัน. TD เป็น dataflow — ทุก node cook ตลอด ไม่มี "ข้ามบรรทัด" → ไม่ branch การรัน แต่ เลือก/กรองว่า data เส้นไหนไหลผ่าน

คิดแบบโค้ดใน TD ทำด้วย
switch / if…elif เลือก 1 จากหลายทางSwitch CHOP/TOP/DAT (index เลือก input) — นี่คือ if/else เวอร์ชัน TD
cond ? a : bSwitch หรือ expression a if cond else b ในช่อง parameter
x > 0.5 → true/falseLogic CHOP (threshold→0/1) · Math/Limit — TD ไม่มี comparison node ตรงๆ

logic ซับซ้อน / state / loopเขียน Python — Script CHOP, CHOP/DAT Execute · if/else จริงอยู่ตรงนี้

ต้องเขียน Python ไหม? ไม่เสมอ — เลือกเส้นทาง/threshold/boolean ง่ายๆ ใช้ node พอ (Switch/Logic) · เขียน Python เมื่อเงื่อนไขซ้อนหลายชั้น มี state หรือต้อง loop

1. ความสำคัญ

Sketch แรกที่ทำเอง — ใช้ ref/export จาก 1.1 ทันที. Feedback loop จะกลับมาใช้ตอน particle trail + drawing accumulation (Day 3 รูปวาดค้างบนจอ = กลไกเดียวกัน). เริ่มด้วยเมาส์ (ยังไม่แตะกล้อง) เพราะ input ตัวเดียว ตัวแปรน้อย ดีบักง่าย — โฟกัส TD logic ก่อน

2. ประเด็นที่ต้องพูด
  • Feedback loop คืออะไร: เอา output ของ "เฟรมก่อน" ป้อนกลับมาผสมกับเฟรมนี้ → ภาพมี "ความจำ" สะสมขึ้นเรื่อยๆ. เทียบ dev: คล้าย state ที่ accumulate ทุก tick แต่ state เป็น "ภาพทั้งภาพ" (texture) ไม่ใช่ตัวเลขเดียว
  • ทำไมต้องมี Composite/Over: ต้องเอา brush อันใหม่ "วางทับ" ภาพเก่าที่ค้างอยู่ — Over = alpha composite (ใหม่บนเก่า). ถ้าไม่ composite จะเห็นแค่ brush ตำแหน่งล่าสุด ไม่ค้าง
  • decay / fade: ก่อนป้อนกลับ คูณภาพด้วยค่า <1 (เช่น 0.95) ทุกเฟรม → เส้นเก่าจางหายช้าๆ. ไม่ลด = ค้างถาวรจอเต็ม · ลดมาก = หางสั้น · ลดน้อย = หางยาว
  • เทคนิคนี้เรียก render-to-texture — ภาพถูกวาดวนกลับเข้าตัวเอง ทำได้ด้วย node แค่ 2–3 ตัว
  • segue: "ตอนนี้ input เป็นเมาส์ — เปลี่ยนเป็นเสียง (1.3) หรือมือ (Day 2) ได้ ด้วยโครงเดิม"

3. เนื้อหา + key scripts (verified · TD 2023.1+ OpenCV 4.8)

Video Device In TOP → Script TOP (ArUco+warp) → out · ตั้ง Script TOP resolution = 1024×1024 · ไฟล์เต็ม + ตัว generate marker อยู่ที่ aruco/ (เปิด/รันได้เลย)

import numpy as np, cv2

DICT = cv2.aruco.getPredefinedDictionary(cv2.aruco.DICT_4X4_50)
_det = cv2.aruco.ArucoDetector(DICT, cv2.aruco.DetectorParameters())   # build ครั้งเดียว
W = H = 1024
DST = np.float32([[0,0],[W-1,0],[W-1,H-1],[0,H-1]])     # TL,TR,BR,BL
_last = np.full((H,W,3), 0.15, np.float32)              # fallback: ภาพ scan ล่าสุด

def order_by_id(corners, ids):                          # คืน [TL,TR,BR,BL] = ID 0,1,2,3 (ใช้ center)
    if ids is None: return None
    c = {int(i): mc.reshape(4,2).mean(0) for mc,i in zip(corners, ids.flatten())}
    if not all(k in c for k in (0,1,2,3)): return None
    return np.float32([c[0],c[1],c[2],c[3]])

def onCook(scriptOp):
    global _last
    src = scriptOp.inputs[0].numpyArray()               # float32 RGBA 0..1, origin ล่างซ้าย
    img = np.flipud(src)                                # flip -> origin บนซ้าย (OpenCV)
    u8  = (np.clip(img[:,:,:3],0,1)*255).astype(np.uint8)
    gray = cv2.cvtColor(u8, cv2.COLOR_RGB2GRAY)
    corners, ids, _ = _det.detectMarkers(gray)
    pts = order_by_id(corners, ids)
    if pts is None:                                     # marker ไม่ครบ 4 -> ค้างภาพล่าสุด
        scriptOp.copyNumpyArray(_alpha(np.flipud(_last))); return
    M = cv2.getPerspectiveTransform(pts, DST)
    warp = cv2.warpPerspective(u8, M, (W,H)).astype(np.float32)/255
    _last = warp
    scriptOp.copyNumpyArray(_alpha(np.flipud(warp)))    # flip กลับ + ใส่ alpha

def _alpha(rgb):
    a = np.ones((*rgb.shape[:2],1), np.float32)
    return np.ascontiguousarray(np.concatenate([rgb,a],2))
